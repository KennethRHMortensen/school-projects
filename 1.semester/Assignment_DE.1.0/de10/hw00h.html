Upon loading the HTML file in the browser, the browser will look through the files and render them in the following order

  1. The browser reads line #1, stating the html document is written in HTML5
  2. The browser reads the <head> tag, and starts rendering each line one by one, checking the metas first
  3. First meta is the utf-8, indicating which character encoding is used
  4. Second meta gives the browser the knowledge of how to control the page's dimensions and scaling, with the attributes of indicating the width should be the device's screen width, along with an indicated zoom-level by default
  5. Next up comes the title of the page, being the name of the page that will be displayed in the page's tab, as well as the browsers tab name at the top
  6. The browser now sees a <link> tag, indicating that theres a reference to either a local (in this case online) link that the page will be fetching something from (in this case a favicon, which is the icon displayed at the left of the title of the page)
  7. Now we see a script tag, which indicates that theres a script needed to run this pages content properly, with a source of its origin. Script tags are used to load script files, such as JS, where link tags are used to link local or online style sheets or frameworks etc.


Now that the <head> tag has been rendered, the browser will move on to the body tag, which is where all the visible content on the page will be visible for the user

  1. Here we see a <h1> tag, meaning it will contain the most important heading tag, as we have heading tags categorizing from 1-6. The <h1> tag has been given an attribute, a class, which makes it accesable from other accompanying files like js or css for editing beyond its origin, in this case its class is used for a javascript.


The javascript file, and what it does

Here we will go through what the JS file does

  1. First of we see the expression 'use strict', indicating that the code should be executed in strict mode
  2. next, we have a variable defined as a 'const', meaning that it can't be changed further on by redefining the variable later on. In this variable, "go" is defined as being a function, which will be triggered later on in the event listener
  3. Within the function "go", we see another variable, defined as let (meaning this can be redefined further on), which is names "targets". This variable makes the browser search through the entire document, for an attribute class with a name of 'title', which is the <h1> text from the html file. This means that from now on, we can access the <h1> by writing "target" in our js code.
  4. next up we see a loop method, defined with a "for" definition. The "for" loop, loops through its code, a number of times. Within this loop, the index at where it should begin, is indicated by defining its first index as "0", meaning the content is a 0-based index. Next within the loop, its written that as long as the index is less than the entire length of our variable "targets", it will continously add 1 to the index, meaning the loop won't stop until the targets length is smaller than the index, which in this case is never.
  5. Now we see that the targets innerHTML, is changed by force through the power of JS, into "Hello, World", meaning it will overwrite the inner html's text, with whatever is written within the JS file, upon rendering this javascript
  6. Now that the function has been defined and written, it is time to render the function. This is done by adding an eventlistener to the browsers window, meaning the browser will listen for an event, and once this has been found, it will load the function named 'go'. This means, that once the browser window is rendered, it will render the function "go", which will change the content of <h1>. This could also be linked to a button, so that the content wouldn't change until a button had been added to the HTML, and upon clicking this, it would change the content...
